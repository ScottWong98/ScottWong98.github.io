本文主要实现了为部署到 GitHub Pages和Coding Pages上的Hexo绑定同一域名。其中国内访问Coding，国外访问GitHub

由于Coding现在已经和腾讯云进行合作，所以本文主要是在腾讯云开发者平台上进行Coding Pages部署的。并且本文是建立在Hexo已经部署到GitHub Pages上的基础之上。

Hexo部署到Coding Pages

新建项目

登录腾讯云开发者平台，进行相应的注册。注册成功后，新建项目，看到如下页面

项目地址必须是yourname.coding.me，yourname就是前面https://dev.tencent.com/u/Syourname/p/中的yourname，项目名称可自定义。点击新建项目按钮。

部署公钥

具体方法和在GitHub中的一样。进入刚才创建的项目，在左侧栏中点击设置中的部署公钥选项。然后点击新建部署公钥按钮，页面如下图

如果你在腾讯云开发者平台上的邮箱和注册GitHub时填写的邮箱一致，那么就找到之前在Github部署公钥时的公钥内容，将其复制到公钥内容那一栏，公钥名称可自定义，记得勾选授予推送权限选项。

然后在终端输入ssh -T git@git.coding.net，如果看到了一句提示信息

说明Coding远程仓库与本地连接。

如果你的邮箱和GitHub中的不一致，请参考此博文：https://www.jianshu.com/p/d21a7ca5b9fb

域名解析

如果你的域名是在阿里云上购买的，找到云解析DNS中的域名解析，对域名进行解析设置。添加A类型

在终端输入ping xxx.coding.me获取你的ipv4地址，输入在记录值中。

和CNAME类型

在记录值中写入你要指向的网址，即你的coding网址。此时，先不要对GitHub进行解析。

可能存在的问题

网址左边的锁不是绿色

在个人博客中，按F12进入调试，当中会显示错误，应该是http的问题，在相应的文件中把http改成https即可
